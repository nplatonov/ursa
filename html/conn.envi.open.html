envi.openR Documentation

envi.open file

Description

envi.open creates object of rmapRaster class, reads ENVI header file and prepares connections for ENVI binary file

Usage

envi.open (fname, resetGrid = FALSE, headerOnly = FALSE, decompress = !headerOnly, ...)

Arguments

fname

Filename; full-name or short-name

resetGrid

Logical. If TRUE then existing base grid (from session.grid) will be overwritten. Otherwise the spatial subsetting will be attempted.

headerOnly

Logical. If TRUE then only reading of ENVI header file without creating connection to binary data; there is no necessary to decompress packed binary in this case. Default is FALSE.

decompress

If ENVI binary file is compressed and you have not to use ENVI values then put decompress=FALSE to avoid useless operation

...

If input file does not exists then these additional arguments will be passed to envi.create function.

Details

envi.open try to find ENVI files (binary and header) and open them. If unsuccessful then function passes ...-arguments to envi.create function

Value

Returns object of class rmapRaster. Values from ENVI binary are not in memory yet.

Author(s)

Nikita Platonov platonov@sevin.ru

See Also

close, envi.create

Examples

session.grid(NULL)
a <- pixelsize()
envi.write(a,"example")
a <- envi.open("example")
dir(pattern="^example.*")
rmap.info(a)
close(a)
rm(a)
envi.remove("example")

## additional arguments are enough to create new ENVI file
dir(pattern="^example.*")
a <- envi.open("example",layername=paste0("test",1:3))
rmap.info(a)
dir(pattern="^example.*")
close(a)
envi.remove("example")
